# Angular Integração Visão geral

Quando integrating Module Federation em um Angular app, choose o guia based no build configuração usado by seu projeto.

| Project tipo | Read next |
| --- | --- |
| Angular CLI project | [Angular CLI Setup](/integrations/framework/angular/angular-cli.html) |
| Nx workspace | [Using Nx CLI for Angular](/integrations/framework/angular/using-nx-for-angular.html) |
| Full micro-frontend exemplo | [Angular MFE](/integrations/framework/angular/angular-mfe.html) |
| SSR scenario | [Angular SSR](/integrations/framework/angular/mf-ssr-angular.html) |
| Service Worker scenario | [Service Workers with Module Federation](/integrations/framework/angular/service-workers-mf.html) |

Angular integrações commonly depend on Angular CLI, uma custom Webpack builder, ou Nx. Antes choosing uma guia, confirm your current build configuração e whether SSR é obrigatório.

Se você apenas need o shared Module Federation configuração concepts, consulte [Configuration](/configure/index.html) for `name`, `remotes`, `exposes`, e `shared`.
